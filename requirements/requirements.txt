EIAnalysis Single-Page Application

Tech Stack: HTMX, Django, SQLite

Requirements:

Initial version (1.0) of the application shall meet the following requirements:
* This shall be a single-page application.
* User shall be able to navigate the local file system and load a JSON text data file into a list of dicts.
* User shall be able to select from a list of statistical analyses to run on the loaded data object (list of dicts).
* The list of statistical analyses shall initially contain just one analysis: Tables Analysis.
* Upon selection of Tables Analysis, the user shall be prompted to select one Outcome Variable and one or more Exposure Variables.
* Executing Tables Analysis shall run TablesAnalysis from the epiinfo package (https://github.com/Epi-Info/epiinfo) and present all results in well-organized tables.
* Tables Analysis results shall include a Summary Table at the top, above the per-variable contingency tables, showing one row per 2×2 exposure variable with the following columns: Variable Name, Odds Ratio (OR), ORLL, ORUL, Risk Ratio, RR LL, and RR UL. The Summary Table shall be initially sorted alphabetically (case-insensitive) by Variable Name. The user shall be able to click any column header to sort the table by that column; clicking the same header again shall reverse the sort direction.
* http://127.0.0.1:8000/ is taken so this application shall use a different localhost URL.
* The application shall be developed using Django and HTMX, with PostgreSQL as the database.

0.3.1 Requirements:
* In the Add/Update Variable Tool, add a new Assignment Type: "Assigned Expression".
* Place "Assigned Expression" above "Difference between Two Dates" in the Assignment Type dropdown.
* Upon selection of "Assigned Expression", the user shall be prompted to enter an expression in a text input field.
* An expression may be an arithmetic expression that returns a number or a logical expression that returns a boolean value.
* An arithmetic expression may contain variable names, literal values, parentheses, and the following operators: +, -, *, /.
* A logical expression may contain variable names, literal values, parentheses, and the following operators: AND, OR, NOT, =, !=, <, <=, >, >=.
* Variable names in expressions shall be case-sensitive and must match exactly the variable names in the dataset
* Literal values in expressions may be strings (enclosed in single or double quotes), numbers, or dates in MM/DD/YYYY format (enclosed in single or double quotes).
* The expression shall be evaluated for each row in the dataset, and the resulting values shall be assigned to the new or existing variable specified by the user.
* If the new variable name specified by the user does not already exist in the dataset, a new variable shall be created with that name and populated with the calculated values. If the new variable name already exists in the dataset, the existing variable shall be updated with the calculated values.
* If an expression cannot be evaluated for a particular row (e.g., due to missing values, invalid syntax, or other errors), the resulting value for that row shall be set to missing/null.
* To test, load Salmonellosis.json and create a new variable called "AteEggs" with the Assigned Expression: ChefSalad = True OR EggSaladSandwich = True. Then run a Frequency analysis on the AteEggs variable and verify that the results match expectations based on the values of ChefSalad and EggSalwich variables in the dataset.
