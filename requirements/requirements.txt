EIAnalysis Single-Page Application

Tech Stack: HTMX, Django, PostgreSQL

Requirements:

Initial version (1.0) of the application shall meet the following requirements:
* This shall be a single-page application.
* User shall be able to navigate the local file system and load a JSON text data file into a list of dicts.
* User shall be able to select from a list of statistical analyses to run on the loaded data object (list of dicts).
* The list of statistical analyses shall initially contain just one analysis: Tables Analysis.
* Upon selection of Tables Analysis, the user shall be prompted to select one Outcome Variable and one or more Exposure Variables.
* Executing Tables Analysis shall run TablesAnalysis from the epiinfo package (https://github.com/Epi-Info/epiinfo) and present all results in well-organized tables.
* Tables Analysis results shall include a Summary Table at the top, above the per-variable contingency tables, showing one row per 2×2 exposure variable with the following columns: Variable Name, Odds Ratio (OR), ORLL, ORUL, Risk Ratio, RR LL, and RR UL. The Summary Table shall be initially sorted alphabetically (case-insensitive) by Variable Name. The user shall be able to click any column header to sort the table by that column; clicking the same header again shall reverse the sort direction.
* http://127.0.0.1:8000/ is taken so this application shall use a different localhost URL.
* The application shall be developed using Django and HTMX, with PostgreSQL as the database.

0.1 Requirements:
* Add Logistic Regression Analysis to the list of available analyses.
* Upon selection of Logistic Regression Analysis, the user shall be prompted to select one Outcome Variable and one or more Exposure Variables.
* User shall have the option to select a Match Variable for matched case-control analysis.
* User shall have the option to select two or more of the Exposure Variables as Interaction Variables.
* Executing Logistic Regression Analysis shall run LogisticRegression from the epiinfo package (https://github.com/Epi-Info/epiinfo) and present results in formats similar to those in the two logistic regression analyses in sample_data\Salmonellosis.html.

0.1.1 Requirements:
* Add Log-Binomial Regression Analysis to the list of available analyses.
* Its user interface shall be nearly identical to that of Logistic Regression Analysis, except that it shall not have a Match Variable option.
* Executing Log-Binomial Regression Analysis shall run LogBinomialRegression from the epiinfo package (https://github.com/Epi-Info/epiinfo) and present results in formats similar to those in the two log-binomial regression analyses in sample_data\Salmonellosis.html.
* Include comparing all calculations to those in the two log-binomial regression analyses in sample_data\Salmonellosis.html as part of testing and validation.

0.1.2 Requirements:
* Add Linear Regression Analysis to the list of available analyses.
* Place it second on the list of analyses, between Tables Analysis and Logistic Regression.
* Its user interface shall be identical to that of Log-Binomial Regression Analysis.
* Executing Linear Regression Analysis shall run LinearRegression from the epiinfo package (https://github.com/Epi-Info/epiinfo) and present results in a format similar to that in the two linear regression analyses in sample_data\Salmonellosis.html.
* Use BabyBloodPressure.json in sample_data to compute statistics to test against those in the two linear regression analyses in sample_data\Salmonellosis.html.

0.1.3 Requirements:
* Add Means Analysis to the list of available analyses.
* Place it second on the list of analyses, between Tables Analysis and Linear Regression.
* User shall be prompted to select a "Means of" variable and an optional "Cross-tabulate" variable.
* Executing Means Analysis shall run Means.Run from the epiinfo package (https://github.com/Epi-Info/epiinfo) and present results in a format similar to that in sample_data\Means.html.
* Use Oswego.json in sample_data to compute means of Age cross-tabulated by Ill and test the results against those in sample_data\Means.html.
* If no Cross-tabulate variable is selected, only the top "Descriptive Statistics" table shall be displayed, with a single row of statistics for the entire dataset.
* If a Cross-tabulate variable with more than two values is selected, the T-Test section of the output shall not be displayed.
